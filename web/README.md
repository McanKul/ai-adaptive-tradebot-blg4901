# Web Control Panel

A local, single-screen web GUI for the trading bot. It is a thin front-end over
the **single entrypoint** `app.py` — every setting you can pass on the command
line is editable from the browser, and "Run" launches the exact same
`python app.py <command> …` invocation, streaming its output back live.

> The form is **generated by introspecting `app.py`'s argparse definitions**, so
> any flag added to the CLI shows up in the UI automatically — there is no
> second list of settings to keep in sync.

## Launch

```bash
# via the single entrypoint (recommended)
python app.py web                 # http://127.0.0.1:8000
python app.py web --port 9000     # custom port
python app.py web --host 0.0.0.0  # expose on the LAN (be careful)

# or directly
python web/server.py              # honours WEB_HOST / WEB_PORT env vars
```

The only extra dependency is Flask (`pip install flask`, already in
`requirements.txt`). The trading commands themselves still need the project's
full dependencies installed (`pip install -r requirements.txt`) and tick data /
API keys configured — the panel runs the command and shows you whatever it
prints, success or error.

## What you can do

- **Run tab** — pick a subcommand (`backtest`, `walk-forward`, `live`,
  `dry-run`, `sweep`, `validate`), fill in any of its flags (grouped into
  Strategy & Market, Sizing & Capital, Exit Rules, Cross-Validation, Sweep
  Filters, …), watch the live command preview, then **Run** and stream the
  output. **Stop** terminates the process (and its children).
  - Blank value fields fall back to the CLI default (shown as placeholder).
  - Boolean flags render as checkboxes; paired flags like
    `--tick-exit/--no-tick-exit` render as a tri-state selector.
  - `--strategy` offers the registered strategies; config/realism/grid path
    fields offer the YAML/JSON files found in the repo.
- **Config Files tab** — load, edit and save any YAML/JSON config the bot reads
  (`live_config*.yaml`, `config/profiles/*`, sweep grids, realism configs, …).
- **Environment tab** — edit the `.env` (Binance / sentiment / Telegram keys).

## Endpoints (for reference)

| Method | Path | Purpose |
|---|---|---|
| GET  | `/api/schema` | Introspected schema of every subcommand & flag |
| GET  | `/api/strategies` | Registered strategy names |
| GET  | `/api/files` | YAML/JSON files in the repo |
| GET/POST | `/api/file` | Load / save a text file (path-jailed to the repo) |
| POST | `/api/preview` | Build the `app.py` command for the current form |
| POST | `/api/run` | Start a run (one at a time) |
| GET  | `/api/stream` | Server-Sent-Events stream of stdout/stderr |
| POST | `/api/stop` | Terminate the running process |
| GET  | `/api/status` | Current run status |

## Security note

This panel executes trading commands and edits files on your machine — it binds
to `127.0.0.1` by default and has **no authentication**. Only expose it beyond
localhost (`--host 0.0.0.0`) on a trusted network, and never on the public
internet.
